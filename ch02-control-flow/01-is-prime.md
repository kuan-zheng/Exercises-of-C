# 判断素数

## 题目

输入一个整数 n，判断它是否为素数（质数）。

## 思考演进

### v1 — 暴力遍历

- 素数定义：大于 1 的自然数，只能被 1 和自身整除
- 最直接的做法：从 2 遍历到 n-1，逐个检查是否能整除
- 时间复杂度：**O(n)**
- 问题：n 很大时效率低

### v2 — 优化到 √n

- 数学性质：若 n = a × b，则 a 和 b 必有一个 ≤ √n
- 只需检查到 `sqrt(n)` 即可，复杂度降为 **O(√n)**
- 边界情况：n ≤ 1 不是素数，n = 2 是素数

### v3 — 跳过偶数

- 除 2 外，所有偶数都不是素数，先排查 n % 2 == 0
- 循环从 3 开始，每次 += 2，只检查奇数因子
- 进一步减少一半的检查次数

## 收获

| 版本 | 时间复杂度 | 关键改进 |
|------|-----------|---------|
| v1 暴力 | O(n) | 基础思路 |
| v2 优化 | O(√n) | 利用配对因子定理 |
| v3 微调 | O(√n/2) | 跳过偶数，常数级优化 |

> 💡 这道题教会我：**数学性质**往往能带来比代码技巧更大的性能提升。
